# Regras do Jogo (RULES)

## PROTOCOLO ZERO (Início de Projeto)
PROIBIDO gerar código imediatamente. Todo novo projeto ou grande funcionalidade DEVE começar obrigatoriamente pela criação de um Artifact de Planejamento (estrutura de pastas, escolha de stack e análise de requisitos). Só inicie o código após aprovação do plano.

1. **Prevenção de Regressão**: NUNCA quebrar código que já funciona.
2. **Leitura Obrigatória**: Sempre ler este arquivo e o PLAN.md antes de começar uma tarefa.
3. **Estilo de Código**:
    - HTML5 Semântico
    - TailwindCSS v3
    - GSAP para animações
4. **Comunicação**: Seja breve, use checklists e fale português.
5. **Modo de Trabalho**: Pense passo a passo. Valide cada etapa antes de ir para a próxima.
6. **REGRA DO CHECKPOINT (Backup)**: Antes de qualquer alteração destrutiva ou grande refatoração, sugira ao usuário fazer um backup manual da pasta ou criar um commit no Git (se estiver ativo). Nunca sobrescreva arquivos críticos sem avisar.
7. **REGRA DO ADVOGADO DO DIABO**: Não obedeça cegamente. Se eu pedir algo que vá quebrar o design, a performance ou a acessibilidade, você tem o DEVER de me alertar e sugerir uma alternativa melhor antes de executar.
